Panduan Perbaikan

MQTT tidak terhubung

  1. 1 Pastikan Anda terkoneksi ke internet dan WiFi stabil
  2. 2 Periksa Broker Host di halaman Pengaturan MQTT — pastikan URL broker benar
  3. 3 Periksa Port WebSocket — untuk HiveMQ Cloud gunakan port 8884
  4. 4 Pastikan Username dan Password sesuai dengan kredensial HiveMQ Cloud
  5. 5 Coba hapus konfigurasi tersimpan dan masukkan ulang di halaman Pengaturan
HiveMQ Cloud menggunakan TLS/SSL wajib. Pastikan browser Anda mendukung WebSocket Secure (WSS).

Sensor tidak membaca data

  1. 1 Pastikan ESP32 menyala dan terhubung ke WiFi (cek LED indikator)
  2. 2 Periksa kabel sensor soil moisture — pin GPIO 34 (ADC)
  3. 3 Periksa kabel sensor DHT22 — pin GPIO 4
  4. 4 Buka Serial Monitor di Arduino IDE (115200 baud) untuk melihat log ESP32
  5. 5 Kalibrasi ulang sensor soil moisture di config.h:
    SOIL_DRY_VALUE (nilai ADC kering) dan SOIL_WET_VALUE (nilai ADC basah)
Capacitive Soil Moisture v2.0: ADC tinggi = kering, ADC rendah = basah. Ukur nilai di udara dan di air untuk kalibrasi.

Pompa tidak merespons

  1. 1 Periksa apakah relay terhubung dengan benar ke pin GPIO 26 (Pompa)
  2. 2 Pastikan relay mendapat daya yang cukup (cek adaptor / power supply)
  3. 3 Cek apakah mode penyiraman otomatis aktif di dashboard
  4. 4 Periksa nilai threshold — pastikan kelembaban tanah memang di bawah batas bawah
  5. 5 Coba kontrol pompa secara manual dari dashboard untuk mengisolasi masalah
Relay module menggunakan Active LOW. LOW = ON, HIGH = OFF. Jika terbalik, ubah RELAY_ACTIVE_LOW di config.h.

Firebase offline / tidak menyimpan data

  1. 1 Pastikan Anda sudah login di dashboard (Firebase Auth)
  2. 2 Periksa koneksi internet Anda
  3. 3 Cek kuota Firebase Realtime Database di Firebase Console
  4. 4 Pastikan Database Rules mengizinkan read/write untuk user yang ter-autentikasi
  5. 5 Periksa konfigurasi API key dan database URL di firebase-config.js
Firebase Spark plan (gratis) memiliki limit 1GB storage dan 10GB/bulan transfer. Jika mendekati limit, pertimbangkan untuk mengurangi frekuensi pengiriman data.

Jadwal penyemprotan tidak berjalan

  1. 1 Pastikan ESP32 terhubung ke internet agar NTP time sync berjalan
  2. 2 Periksa timezone di config.h — harus GMT_OFFSET_SEC = 25200 untuk WIB (UTC+7)
  3. 3 Pastikan jadwal sudah dikirim ke ESP32 melalui MQTT (cek log di dashboard)
  4. 4 Cek apakah jadwal tersimpan di NVS (restart ESP32 dan lihat Serial Monitor)
  5. 5 Pastikan hari yang dipilih di jadwal sesuai dengan hari saat ini
NTP server default: pool.ntp.org. Jadwal hanya akan berjalan jika waktu NTP sudah tersinkronisasi.
